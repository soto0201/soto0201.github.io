自分の研究について

先日のゼミでの出来事

研究の進捗がそんなに良くなく、予定よりもうまく進んでいない感じです…
そもそも、研究に必要な技術要素について自分があまり多くを知らないのが原因のような気がしています。
というより思っていた以上に難しかったので、ゼミでこんな会話がありました。

ぼく「kono先生、この研究難しくないですか？」
先生「それはそう。先輩(ryokkaさん)が3年間でできなかった事を1年以内にやろうとしているのだから」
ぼく「なるほど(白目)」

先行き不安ですね。

研究に必要そうな技術要素

前述した通り、私の今の研究内容はほとんどryokkaさんと同じです。
ryokkaさんの修士論文は Hoare Logic を使用したCbCの検証をしており、Loopの検証までは完成していました。
ただ、Red-Black Tree の検証ができなかったそうです。ここの完成が自分の研究になっています。
時間があれば絶対完成させられる先輩なので、2月後半は私に研究の引き継ぎをしていたので時間が足りなかったのでしょう…
kono先生も難しいと言っていたので、これができる先輩本当にすごいと思います…

ここで、必要になる技術要素をあげると以下のようになるはず

  • CbC
    • CbCの概念や構成要素など(Coda Segmentとか…ここら辺も理解しないといけない)
    • Gears OS についての理解(書いたagdaをここで動かすため)
  • agda
    • 基本的なagdaの記述方法(こいつはまだ良い)
    • 自然演繹やHoare Logicなどの形式論理(これの理解が一番足りない感じ)

これらが必要なはず。書くと簡単に見えますが、一つ一つの内容が濃いんですよね…
見てわかる通り、3年次までに習ってきたことと関連性があまりない感じになっています。
agdaは2年次の際に習ったはずなのですが結構理解が足りていない感じです。
まず研究に必要な要素から調べていく感じになっていたので、出だしは他の人よりも遅れる感じになるのも頷ける…のでしょうか？

最近はこの研究の進め方にも慣れてきたのと、4月からずっとメンタルやっていたのが最近は回復してきた感じなので、
やることは多いですが、新しい事を1から始めるのも面白いのでいつも通りちまちま進めていきたいと思っています。
6月2週までに中間報告の雛形作れるかな…密かに研究進度の目安にしています。

今日は

上述の Hoare Logic を提唱した論文の、
An Axiomatic Basis for Computer Programming (Charles Antony Richard Hoare著) についての記事を書けるといいですね…
論文で使用している論理の提唱元を知らないのは話にならないので、ひとまずはこれを読もう！という意気込みです。
英語難しいです…
